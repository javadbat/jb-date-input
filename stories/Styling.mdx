<Title>Styling</Title>

`JBDateInput` is a composed component. The date input host owns the calendar trigger and popover shell variables, while the inner `JBInput` and `JBCalendar` keep their own reusable styling recipes and APIs.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Date-input recipes should use those common tokens where possible, then compose the existing `JBInput` and `JBCalendar` recipes instead of copying their variable blocks.

## Styling Model

Use four layers when styling date-input examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-date-input/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-date-input/stories/styles).
- Input recipes are reused from [`modules/jb-input/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-input/stories/styles).
- Calendar recipes are reused from [`modules/jb-calendar/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-calendar/stories/styles).
- Public date-input variables use `--jb-date-input-*` on the host and `--jb-popover-*` on `::part(popover)`. Use [JBInput Styling](../../jb-input/stories/Styling.mdx) and [JBCalendar Styling](../../jb-calendar/stories/Styling.mdx) as the source of truth for child component variables.

Each example uses one shared example theme pack from [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles), one existing `JBInput` recipe, one existing `JBCalendar` recipe, and one small date-input-specific popover shell recipe. The style story applies the child recipe classes to the internal `jb-input` and `jb-calendar` instances for the demo.

## Recommended Customization Path

1. Use `--jb-date-input-*` variables first for the host margin and calendar trigger icon.
2. Reuse an existing `JBInput` recipe for the text-field layer. See [JBInput Styling](../../jb-input/stories/Styling.mdx) for the full input API.
3. Style the picker shell by setting `--jb-popover-*` variables on `jb-date-input::part(popover)`.
4. Reuse an existing `JBCalendar` recipe for the picker content. See [JBCalendar Styling](../../jb-calendar/stories/Styling.mdx) for the full calendar API.
5. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
6. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
7. Add a new date-input variable only when a needed internal surface cannot be customized with current public variables or official parts.

## Starter Recipe

Use this minimal recipe when a product only needs brand alignment across the date-input host and popover shell. Use the linked child component styling pages for the inner input and calendar.

```css
jb-date-input.brand-date-input {
  --jb-date-input-calendar-icon-color: #2563eb;
  --jb-date-input-calendar-icon-color-active: #1d4ed8;
}

jb-date-input.brand-date-input::part(popover) {
  --jb-popover-bg-color: #ffffff;
  --jb-popover-border-radius: 1rem;
  --jb-popover-padding: 1rem;
  --jb-popover-z-index: 1000;
}
```

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles). Composed components should reuse child component recipes instead of copying their variables into the parent recipe.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-input/stories/styles/style-carbon.css */
.theme-name-input {
  /* reusable jb-input recipe */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-calendar/stories/styles/style-carbon.css */
.theme-name-calendar {
  /* reusable jb-calendar recipe */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-date-input/stories/styles/style-carbon.css */
.theme-name-date-input {
  /* date-input host variables */
}

.theme-name-date-input::part(popover) {
  /* jb-popover background, radius, padding, and shell treatment */
}

.theme-name-date-input::part(calendar) {
  /* surface reset when the popover owns background and shadow */
}
```

This keeps common theme tokens reusable, keeps child component recipes as the source of truth, and leaves `JBDateInput` responsible only for the host trigger and popover shell.
